Hexo+Github Pages 搭建个人博客

发表于 2016-12-04

Hexo 是一个快速、简洁且高效的博客框架。– Hexo官网

学习Hexo之前，可以先了解一些常用的命令 点击查看

1.系统环境配置

下载安装 Node.js 和 git

git

node.js

git 教程个人推荐 廖雪峰 的教程

2.安装Hexo

打开 git bash 面板，安装 Hexo：

npm install –g hexo-cli

Windows系统下安装完成后会显示一些 warning ，可以忽略不管。

查看本地Hexo版本（最好安装3.0以上版本）：

hexo version

如果显示出Hexo版本说明Hexo安装成功。

3.搭建本地博客

在本地新建一个文件夹 blog (文件命名随意)，在blog文件夹下右键打开 git bash 面板，输入以下命令自动安装相应的文件：

hexo init

等待安装完成后，blog文件夹下会生成如下图所示的文件：

目录简介：

node_moduled  // hexo的模块管理，可自行忽略
scaffolds   // 模板文件夹，用户可以自定义模板
source      // 资源文件夹是存放用户资源的地方,用户的文章放在 _posts 文件夹下
themes      // 主题文件夹，用户可以根据自己的需求安装主题
_config.yml // hexo全局配置文件（用户可以根据自己的需求进行个性化配置）

安装启动服务器插件：

npm install hexo-server --save

启动服务（也可以简写为：hexo s），默认端口为 4000：

hexo server

输入 http://localhost:4000 查看效果

ctrl+c 停止服务

到此为止，本地博客已经搭建完成。

4.将本地博客部署到Github Pages

在使用之前需要先了解下什么是 Github 和 Github Pages。Github 被称为全球最大的社交编程及代码托管平台，上面有大量的开源代码，广为开发者喜爱。Github Pages 是Github 公司提供的免费的静态网站托管服务，并且用户可以绑定自己独立的域名。没有Github账号的用户需要去 官网注册（这里不详细介绍）。

注意： 新用户需要在 settings 配置SSH keys(这里不详细介绍)。

进入 github 主页，选择 new repository，进入到 新建仓库 的界面。新建一个 USERNAME.github.io 的仓库，这里 USERNAME 是 Github 的用户名，且只能是 Github 的用户名。Github Pages搭建博客已经完成，下面把本地的 Hexo 博客托管到Github上。

打开blog文件夹下的 _config.yml 文件（建议用Nodepad++或者记事本打开），找到文件最下面的deploy:进行配置修改。

deploy:
  type: git
  repository: git@github.com:winlily/winlily.github.io.git
  branch: master

注意： yaml语法有非常严格的要求，“：” 后面一定要加上空格，并且缩进也要一致，最初配置时因为缩进没有一致出现了错误如下：

如图所示右上角有一个切换按钮，Use HTTPS 点击后会切换到 HTTPS 链接如下：

到此配置已经完成，现在需要用命令部署到Github，部署到 Github 前需要在本地生成

hexo g

部署到 Github

hexo d

现在访问 USERNAME.github.io 即可预览自己的博客。

5.Github Pages 绑定自己独立的域名

当然，这一步的前提是要有个个人独立域名。作为一枚学生党的我首先还是考虑了下价格[haha] ,这一点阿里云和 腾讯云针对学生党都有些优惠，相比之下我个人选择的是后者。

有了域名，现在我们要把它用起来。

  • 首先，进入到域名管理界面，进行域名解析；
  • 其次，我们需要 ping 出 winlily.github.io的IP地址；

  • 再者，在域名管理界面添加两个记录：

    1）记录类型A 主机记录为www 记录值为winlily.github.io的IP地址；

    2）记录类型为CNAME 主机记录为@ 记录值为winlily.github.io

  • 最后，在winlily.github.io项目根目录下建文件，文件名为CNAME(没有后缀)，内容写自己的域名地址（winlily.me）。

现在输入自己的域名即可访问自己的博客，到此Hexo+Github Pages 搭建个人博客全部完成。
